# GET /api/serviceorder/servantlist

服务者查询自己的订单列表。

**前缀**：`/api/`（需要登录）

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | int | 否 | 状态：`0`=未生效, `1`=待接单, `10`=进行中, `20`=已完成, `90`=已取消 |
| from | int64 | 否 | 分页游标（上一批次最大 ID，按 id 倒序） |

## 响应

`ServiceOrderDetail[]`。订单中 `servant_user_id` 为当前登录服务者。

## 常见用法

- 查待接单：`?status=1` — 决定是否接单
- 查进行中：`?status=10` — 今日 / 近期待履约
- 查已完成：`?status=20` — 结算对账
