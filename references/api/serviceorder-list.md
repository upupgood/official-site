# GET /api/serviceorder/list

客户查询自己的订单列表。

**前缀**：`/api/`（需要登录）

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | int | 否 | 状态：`0`=未生效, `1`=待接单, `10`=进行中, `20`=已完成, `90`=已取消 |
| from | int64 | 否 | 分页游标（上一批次最小 ID，按 id 倒序） |
| ids | string | 否 | 逗号分隔订单 ID 列表（最多 50 个）；传入时忽略 `status` 和 `from` |

## 响应

`ServiceOrderDetail[]`（结构见 `SKILL-CUSTOMER.md` 数据结构章节）。

## 常见用法

- 查进行中订单：`?status=10`
- 查已完成订单：`?status=20`
- 批量查特定订单：`?ids=123,456,789`
