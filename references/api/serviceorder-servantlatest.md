# GET /api/serviceorder/servantlatest

服务者查询最新服务单（按服务时间最接近排序）。

**前缀**：`/api/`（需要登录）

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | int | 否 | 状态：`0`=未生效, `1`=待接单, `10`=进行中, `20`=已完成, `90`=已取消 |
| finished | bool | 否 | 是否已完成（待客户确认） |

## 响应

`ServiceOrderDetail[]`，按服务时间最接近当前时间排序。

## 用途

- 回答 "下一单什么时候？"
- 回答 "我最近要做的单在哪里？"
- 展示 "今日日程" 卡片

## 与 `/api/serviceorder/servantlist` 的差异

| 维度 | servantlatest | servantlist |
|------|--------------|-------------|
| 排序 | 按服务时间最接近 | 按 ID 倒序 |
| 适用场景 | "下一单是什么" | "我的订单历史" |
