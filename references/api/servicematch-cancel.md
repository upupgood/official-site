# POST /api/servicematch/cancel

服务者取消已报名的需求。

**前缀**：`/api/`（需要登录）

## 请求体

```json
{ "req_id": 12345 }
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| req_id | int64 | 是 | 需求 ID |

## 响应

HTTP 200 即成功。

## 约束

- **仅需求 `status=1`（进行中）时允许取消**
- 需求已接单（`status=2`）后不可取消报名，对应订单已生成
- 取消后服务者从该需求的匹配列表中移除
