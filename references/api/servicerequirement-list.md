# GET /api/servicerequirement/list

查询当前客户自己的服务需求列表。

**前缀**：`/api/`（需要登录）

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | int | 否 | 状态筛选：`0`=未提交, `1`=进行中, `2`=已接单, `3`=已完成, `9`=已取消 |
| from | int64 | 否 | 分页游标（上一批次最小 ID，按 id 倒序） |

## 响应

`ServiceRequirementInfo[]`（结构见 `SKILL-CUSTOMER.md` 数据结构章节）。

## 状态说明

| status | 含义 |
|--------|------|
| 0 | 未提交（草稿） |
| 1 | 进行中（已提交，匹配中） |
| 2 | 已接单（已生成订单） |
| 3 | 已完成 |
| 9 | 已取消 |
